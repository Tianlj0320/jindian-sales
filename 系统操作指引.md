# 金典软装ERP V4.0 — 系统操作指引

## 一、业务流程总览

```
创建订单(initial) → 已量尺(measured) → 已确认(confirmed)
  → 采购拆分(split) → 采购中(purchasing)
  → 全部到货/已入库(stocked) → ⭐自动生成加工单
  → 加工中(processing) → 已完成(completed)
  → 已排安装(install_scheduled) → 已安装(installed) → 已验收(accepted)
```

## 二、模块操作说明

### 1. 订单管理

| 操作 | 路径 | 说明 |
|------|------|------|
| 新建订单 | `/orders/new` | 填写客户信息、选择产品（面料/辅料/成品），设置采购类型 |
| 编辑订单 | 订单列表 → 点击编辑 | 仅 initial 状态可编辑 |
| 推进 | 订单详情 → 推进按钮 | 按流程向下一状态推进（自动校验前置条件） |
| 回退 | 订单详情 → 回退按钮 | 回退到之前状态（关联待采购PO会被自动删除） |
| 采购拆分 | 订单推进到 confirmed 后自动触发 | 按供应商/采购类型分组生成采购单 |

**采购类型说明**：
- **物料**：面料类，需经加工车间制作 → 会出现在加工单中
- **成品**：供应商直接交付成品 → 不进入加工流程，采购单显示尺寸+工艺
- **辅料**：配件类 → 不进入加工流程

### 2. 采购管理

| 操作 | 路径 | 说明 |
|------|------|------|
| 采购拆分预览 | 推进订单 → 自动弹出 | 查看按供应商分组的采购方案 |
| 生成采购单 | 确认拆分方案后 | 系统自动生成PO编号（PO20260514xxx） |
| 收货 | 采购单详情 → 收货按钮 | 输入实际到货数量，系统自动计算剩余待收 |
| **批量收货** | 采购跟踪标签页 → 一键全部收货 | 勾选多个PO同时收货 |
| 收货回退 | 采购单详情 → 回退按钮 | 选择仓库，回退入库操作 |

**自动功能**：当某订单所有PO全部到货时，系统自动：
1. 将订单状态推进到「已入库」
2. **自动生成加工单**（针对物料类型的明细）
3. 将订单状态继续推进到「加工中」
4. 加工单列表立即可见

## 三、权限系统说明

### 权限级别

系统预置4个角色，每个角色拥有不同的操作权限：

| 角色 | 权限 | 适用人员 |
|------|------|---------|
| admin（超级管理员） | 所有权限 `[*]` | 老板/系统管理员 |
| manager（店长） | 订单/采购/仓库/加工/安装/财务/报表 | 店长/经理 |
| staff（导购） | 工作台/订单/客户/产品 | 导购/销售员 |
| installer（安装工） | 仅安装管理 | 安装师傅 |

### 权限控制逻辑

- **后端**：关键操作（库存调整、角色管理、员工管理）需要对应权限
- **前端菜单**：根据权限自动显示/隐藏菜单项
- **路由守卫**：直接访问无权页面会自动跳转工作台

### 权限配置

1. 登录 admin 账号（admin / admin123）
2. 进入「基础资料 → 角色权限」
3. 点击编辑角色的权限复选框
4. 保存后对应用户自动生效

## 四、新功能与改进

### V4.0 新增功能

| 功能 | 说明 |
|------|------|
| 采购收货自动生成加工单 | 全部到货后系统自动创建加工单，无需手动操作 |
| 库存手动调整 | 仓库页面可手动调整库存数量（权限：warehouse） |
| 批量收货 | 采购跟踪页支持多PO同时收货 |
| 权限控制系统 | 完整的角色权限管理，菜单/路由按权限过滤 |
| 采购单内容压缩 | 采购单详情只显示核心字段，成品显示尺寸+工艺 |
| 打印预览自适应 | 所有打印预览窗口自适应宽度 |

### 操作流程变化

**旧流程**：收货 → 手动去加工页面点生成 → 才能在加工列表看到
**新流程**：收货 → 系统自动生成 → 加工列表直接可见

## 五、常见问题

**Q: 加工页面查不到订单？**
A: V4.0 已自动处理——采购单全部到货后系统会自动生成加工单。如果仍然看不到，可能是：
- 订单没有「物料」类型的明细（只有成品/辅料不需要加工）
- 检查加工单页面的筛选条件是否重置

**Q: 库存调整提示无权限？**
A: 需要 `warehouse` 权限，请联系管理员在「角色权限」中配置

**Q: 如何删除错误的采购单？**
A: 在订单管理中回退订单到「已确认」之前的状态，关联的「待采购」PO会被自动删除

**Q: 为什么采购单不显示金额？**
A: 采购单页面已按业务要求隐藏金额信息（单价/小计/总金额），仅保留采购核心信息
