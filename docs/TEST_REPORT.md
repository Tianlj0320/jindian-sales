## 三、缺陷列表

### 缺陷 #1
- **模块：** 文档 / 冒烟测试脚本
- **描述：** 冒烟测试用例中测试账号密码错误
- **复现步骤：** 运行 `python tests/test_api.py`，使用账号 13900001111/jd8888 登录失败
- **预期 vs 实际：** 期望 jd8888 可以登录，实际该账号（13900001111）密码为 123456
- **严重程度：** P2（冒烟测试脚本需修正）

---

### 缺陷 #2
- **模块：** 订单列表 API
- **描述：** `/api/orders` 列表接口响应结构不一致——有时返回 `{"success": true, "data": {"items": [...]}}`，有时直接返回 `{"success": true, "items": [...]}`
- **复现步骤：** 调用 `GET /api/orders?page=1&page_size=10`，响应中 data 为空但字段在顶层
- **预期 vs 实际：** 期望统一返回 `{"success": true, "data": {"items": [...], "total": N}}`，实际部分响应 `data` 为空对象
- **严重程度：** P1（API 契约不一致，影响前端解析）

---

### 缺陷 #3
- **模块：** 订单状态流转
- **描述：** 订单状态推进到 `completed` 后未自动生成安装单，最终状态应为 `install_order_generated`
- **复现步骤：** 创建订单 → 连续调用 7 次 `/api/orders/{id}/advance`，最终状态为 `completed` 而非 `install_order_generated`
- **预期 vs 实际：** 期望第 7 次推进后 `status_key == "install_order_generated"`，实际返回 `"completed"`
- **严重程度：** P1（V3.0 核心功能：安装单自动生成未实现）

---

### 缺陷 #4
- **模块：** 客户创建 API
- **描述：** 创建客户成功后响应中 `data` 字段仅包含 `id` 和 `code`，不包含 `name`、`phone` 等创建字段
- **复现步骤：** POST `/api/customers` 创建客户后查看响应 `body["data"]`
- **预期 vs 实际：** 期望 `data` 包含完整客户信息，实际只有 `{id, code}`
- **严重程度：** P2（返回信息不完整）

---

### 缺陷 #5
- **模块：** 客户手机号查重
- **描述：** 系统允许创建手机号重复的客户，未做唯一性校验
- **复现步骤：** POST `/api/customers` 创建手机号 A → 再次 POST 相同手机号 A，第二次也成功
- **预期 vs 实际：** 期望第二次创建失败（手机号已被注册），实际两次均返回 `success: true`
- **严重程度：** P0（数据完整性违规）

---

### 缺陷 #6
- **模块：** 客户更新 API
- **描述：** 更新客户信息后响应 `data` 字段为空对象 `{}`，无法确认更新结果
- **复现步骤：** PUT `/api/customers/{id}` 更新客户信息后查看响应
- **预期 vs 实际：** 期望 `data` 包含更新后的完整客户信息，实际 `data: {}`
- **严重程度：** P2（响应信息不完整）
