## 二、采购单拆分 API

**接口：** `POST /api/purchase-orders/split/{order_id}`

**触发时机：** 订单状态变为「已确认」后，由系统自动调用或人工触发。

**核心逻辑：**
- 按 `supplier_id` 分组，每组生成一张采购单
- 同一供应商 + 多张订单 + 交期±3天 → 合并成一张采购单
- 同一供应商 + 同一订单 + 不同交期 → 分开生成
- 同一供应商 + 交期差>3天 → 分开生成

---

### TC-PO-SPLIT-001：正常拆分 — 订单含多供应商物料

| 字段 | 内容 |
|------|------|
| **用例名称** | 正常拆分_多供应商物料 |
| **API 路径** | `/api/purchase-orders/split/{order_id}` |
| **HTTP 方法** | POST |
| **前置条件** | 存在一张已确认订单（order_id=1），含 3 个不同供应商的物料 |
| **输入参数** | `order_id = 1` |
| **预期输出** | - 返回拆分后的采购单列表<br>- 生成 3 张采购单（每供应商各 1 张）<br>- 每张采购单关联 order_id=1<br>- 响应码 200 |
| **测试类型** | 正常 |

---
