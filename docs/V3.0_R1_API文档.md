## 一、采购单（purchase_orders）

### 1.1 订单拆分采购单
```
POST /api/purchase-orders/split/{order_id}
```
**说明**：核心接口。订单确认后调用，按 supplier_id 分组生成采购单。

**请求**：
- `order_id`（路径参数）：订单ID
- Header: `Authorization: Bearer <token>`

**拆分逻辑**：
1. 读取订单明细（order_items），按 `supplier_id` 分组
2. 同一供应商 → 一张采购单
3. 无 supplier_id → 归入"未分配供应商"组
4. PO 号规则：`PO` + 年月日 + 3位序号

**响应**：
```json
{
  "success": true,
  "message": "成功拆分为 3 张采购单",
  "purchase_orders": [
    {
      "id": 1,
      "po_no": "PO20260421001",
      "supplier_name": "供应商A",
      "total_amount": 2500.00,
      "status": "待采购",
      "expected_date": "2026-04-28",
      "item_count": 5
    }
  ]
}
```

**错误码**：
- `404`：订单不存在
- `400`：订单不可重复拆分

### 1.2 采购单列表
```
GET /api/purchase-orders?status=&keyword=&page=1&page_size=20
```
**说明**：分页展示采购单列表。

**请求参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| status | string | 否 | 状态筛选：待采购/已下单/部分到货/全部到货 |
| keyword | string | 否 | 搜索 PO 单号或供应商名 |
| page | int | 否 | 页码，默认1 |
| page_size | int | 否 | 每页条数，默认20，最大100 |

**响应**：`{ success, total, page, page_size, items: [...] }`

### 1.4 更新采购单状态
```
PATCH /api/purchase-orders/{po_id}
```
**说明**：更新采购单状态、物料、日期等。

**请求 Body**：
```json
{
  "status": "已下单",
  "remark": "已向供应商下单",
  "expected_date": "2026-05-01",
  "arrived_date": "2026-05-03",
  "items": [
    {"product_id": 1, "product_code": "P001", "product_name": "窗帘布A款",
     "spec": "280x200", "qty": 5, "unit_price": 120.00, "amount": 600.00}
  ]
}
```

**状态枚举**：`待采购` → `已下单` → `部分到货` → `全部到货` → `已完成`

### 1.5 合并采购单
```
POST /api/purchase-orders/merge
```
**说明**：将多张同供应商采购单合并为一张。

**请求 Body**：
```json
{ "po_ids": [1, 2, 3] }
```

**条件**：至少2张，且必须同供应商。

**响应**：
```json
{
  "success": true,
  "message": "成功合并 3 张采购单",
  "purchase_order": {
    "id": 10,
    "po_no": "PO20260421010",
    "supplier_name": "供应商A",
    "total_amount": 5000.00,
    "item_count": 12
  }
}
```

## 二、生产反馈（production_feedback）

### 2.1 创建生产反馈
```
POST /api/production-feedback
```
**说明**：生产中发现问题（质量/残次/米数不足）时创建反馈单。

**请求 Body**：
```json
{
  "order_id": 1,
  "purchase_order_id": 5,
  "feedback_type": "quality",
  "description": "面料有色差，部分位置颜色明显偏深",
  "photos": ["https://example.com/photo1.jpg"]
}
```

**feedback_type 枚举**：
| 值 | 说明 |
|----|------|
| `quality` | 面料质量问题 |
| `defect` | 面料残次 |
| `shortage` | 米数不足 |

**响应**：
```json
{
  "success": true,
  "data": {
    "id": 1,
    "feedback_no": "FB20260421001",
    "feedback_type_label": "质量"
  }
}
```

---

### 3.3 手动创建安装单
```
POST /api/installation-orders
```
**说明**：通常由生产完成后自动生成，也可手动创建。

**请求 Body**：
```json
{
  "order_id": 1,
  "scheduled_date": "2026-05-01",
  "installer_id": 2,
  "install_time_slot": "09:00-12:00",
  "install_requirements": "客户要求周末安装，注意不要弄脏墙壁",
  "receivable_amount": 300.00,
  "remark": ""
}
```

**响应**：
```json
{
  "success": true,
  "data": {
    "id": 1,
    "ins_no": "INS20260421001",
    "status": "待分配"
  }
}
```

---

### 3.4 更新安装单（核心操作）
```
PATCH /api/installation-orders/{ins_id}
```
**说明**：分配安装工、确认安装、验收等全流程操作。

**请求 Body**：
```json
{
  "status": "已分配",
  "installer_id": 2,
  "scheduled_date": "2026-05-01",
  "install_time_slot": "09:00-12:00",
  "install_photo": ["https://example.com/before.jpg", "https://example.com/after.jpg"],
  "customer_signature": "base64签名数据",
  "received_amount": 300.00,
  "remark": ""
}
```

**status 枚举**：`待分配` → `已分配` → `已安装` → `已验收`

**验收后联动**：status=已验收 时，关联订单状态自动更新为"已验收"。

---

### 3.5 自动生成安装单
```
POST /api/installation-orders/auto-generate/{order_id}
```
**说明**：生产完成后调用，自动根据订单信息创建安装单。

**触发时机**：订单状态变为"已完成"/"生产完成"时。

**响应**：
```json
{
  "success": true,
  "data": {
    "id": 1,
    "ins_no": "INS20260421001",
    "status": "待分配"
  }
}
```

**错误**：若订单已有安装单，返回错误（防止重复生成）。

## 五、V3.0 订单状态流转

```
已创建 → 已确认 → 【已拆分】→ 采购中 → 已到货 → 生产中
                                              ↓
                                    ┌─────────┴─────────┐
                                    ↓                   ↓
                              生产正常            【生产异常】
                                                      ↓
                                              【生产反馈】→ 补货/调货
                                                      ↓
                                               继续生产 → 已完成
                                                         ↓
                                              【安装单已生成】
                                                         ↓
                                                    已发货 → 已安装 → 已验收 → 订单完结
```
