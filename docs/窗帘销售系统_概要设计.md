### 1.2 技术栈

| 组件 | 选型 | 说明 |
|------|------|------|
| 后端框架 | FastAPI + Uvicorn | 异步高性能， 自动 OpenAPI 文档 |
| ORM | SQLAlchemy 2.0 | Flask-SQLAlchemy 风格 |
| 数据库 | SQLite | 轻量无需运维，够用5年 |
| 前端框架 | Vue 3（CDN，Global build）| 无需 Node.js 构建，SEO 差但内部工具无所谓 |
| UI 组件 | Element Plus（CDN）| 按需引入部分组件 |

| 部署 | Systemd 用户服务 |开机自启，崩溃自恢复 |

## 3. API 设计

### 3.1 认证机制

所有 `/api/*` 接口（除 `/api/auth/login`、`/api/track`、`/api/sms/send`）需要 Header：
```
Authorization: Bearer <jwt_token>
```

JWT Payload:
```json
{
  "sub": "1",          // employee.id
  "name": "韩霜",
  "role": "老板",
  "exp": 1776844958    // 72小时后过期
}
```

### 3.2 统一响应格式

```json
// 成功
{ "success": true, "data": {...} }

// 分页
{ "success": true, "total": 100, "items": [...] }

// 错误
{ "detail": "错误描述" }
```

### 3.3 主要 API 列表

**认证**
- `POST /api/auth/login` — 手机号+密码登录
- `GET /api/auth/me` — 当前登录用户信息

**首页**
- `GET /api/dashboard` — KPI卡片数据（今日订单/月销售/逾期/待安装）

**订单**
- `GET /api/orders` — 列表（支持 page/page_size/year/month/status 筛选）
- `POST /api/orders` — 新建
- `PUT /api/orders/{id}` — 编辑
- `PUT /api/orders/{id}/status` — 变更状态（含历史记录）
- `GET /api/orders/{id}` — 详情

**客户**
- `GET /api/customers` — 列表
- `POST /api/customers` — 新建
- `PUT /api/customers/{id}` — 编辑
- `DELETE /api/customers/{id}` — 删除

**产品/供应商/布版**
- `GET /api/products` — 产品列表
- `POST /api/products` — 新建产品
- `GET /api/products/suppliers` — 供应商列表
- `POST /api/products/suppliers` — 新建供应商
- `GET /api/products/categories` — 布版分类
- `POST /api/products/categories` — 新建布版

**员工**
- `GET /api/employees` — 列表
- `POST /api/employees` — 新建
- `PUT /api/employees/{id}` — 编辑

**报表**
- `GET /api/reports/sales` — 月度销售（year/month）
- `GET /api/reports/trend` — 销售趋势（最近12月）
- `GET /api/reports/employee-performance` — 员工业绩
- `GET /api/reports/product-rank` — 产品排行

**打印**
- `GET /api/print/{template}/{order_id}` — HTML打印模板（contract/measurement/processing/install）

**客户查进度**
- `GET /api/track?order_no=...&phone=...` — 扫码查进度

**安装工**
- `POST /api/installer/login` — 安装工短信登录
- `GET /api/installer/tasks` — 任务列表
- `POST /api/installer/tasks/{id}/complete` — 确认完成

## 4. 前端架构（index.html）

### 4.1 页面路由（Hash 模式）

```
#dashboard  ─ 首页工作台
#orders     ─ 订单列表 / 新建订单
#customers  ─ 客户资料
#products   ─ 产品资料
#suppliers  ─ 供应商管理
#employee   ─ 员工资料
#dict       ─ 码表管理
#purchase   ─ 采购管理
#warehouse  ─ 仓库管理
#finance    ─ 财务结算
#reports    ─ 统计报表
```

### 4.2 状态管理

纯 Vue 3 Composition API + reactive/ref，页面级别状态，无全局 Store：

```javascript
const currentUser = ref(null)
const page = ref('dashboard')     // 当前页面
const showLogin = ref(true)       // 登录弹窗
const dlgXXX = ref(false)         // 各弹窗显隐
```

### 4.3 关键设计决策

1. **Vue CDN 而非工程化**：店员电脑无需 Node.js 环境，HTML 文件拷贝即用
2. **el-table 替换为原生表格**：Element Plus colspan 计算 bug 无解
3. **打印用 HTML 而非 PDF**：浏览器 `window.print()`，无需后端 PDF 生成
4. **SQLite 而非 PostgreSQL**：月均100单数据量极小，SQLite 完全够用，运维零成本

## 6. 安全设计

| 威胁 | 防护 |
|------|------|
| 密码泄露 | bcrypt 哈希 |
| Token 伪造 | JWT HMAC 签名（密钥在环境变量） |
| SQL 注入 | SQLAlchemy 参数化查询 |
| XSS | 前端 innerHTML 显示用户输入（内部系统，风险可控） |
| CSRF | 内部系统，仅局域网访问 |
| 暴力破解 | JWT 72小时自动过期 |

## 7. 已知局限与未来升级路径

| 局限 | 说明 | 升级方案 |
|------|------|---------|
| SQLite 并发写入 | 多店员同时开单可能锁DB | 迁移 PostgreSQL |
| 无数据权限 | 所有登录员工看到全部数据 | 增加 role 权限体系 |
| 前端代码全在一文件 | 1663行，维护成本上升 | 拆分 Vue 单文件组件 |
| 无操作审计日志 | 谁在什么时候改了什么单 | 增加 audit_log 表 |
| 打印模板不可配置 | 硬编码 HTML | 数据库存储模板 |
