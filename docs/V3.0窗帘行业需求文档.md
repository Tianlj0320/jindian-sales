## 二、行业软件研究结论

### 2.1 主流窗帘/软装软件功能模块

| 软件 | 核心功能 |
|------|----------|
| 宏达窗帘布艺业务管理系统 | 材料管理、订单管理、财务管理、采购/耗损/退货、库存明细 |
| 简道云（软装行业版） | 库存管理、客户跟踪、多门店、移动下单/测量、照片验收、API对接 |
| 百惠经营管理系统 | 多计算方式（按米/按平方/按窗）、报价单打印 |
| Softex/Madrigal | 生产计划、质量监控、订单追踪、客户CRM |
| 帘老板APP | 商品浏览、测量预约、上门安装、订单物流跟踪 |

### 2.2 窗帘行业特性关键发现

1. **多种计价方式**：按米、按平方、按窗、固定价
2. **测量→生产→安装全流程**：需要贯穿整个链路
3. **面料幅宽计算**：不同幅宽面料用量不同，需要自动算料
4. **辅料复杂**：轨道、罗马杆、挂钩、绑带、铅坠等
5. **多门店/多员工**：导购、测量工、安装工、财务不同角色
6. **照片记录**：测量照片、安装前后对比
7. **微信通知**：客户实时知道进度

## 三、V3.0 新增功能需求

### 3.1 业务流程重构（全链路走通）

```
获客 → 测量 → 报价 → 签单 → 【订单确认】→ 生产 → 送货 → 安装 → 售后 → 复购
                              │
                              ↓
                        【采购拆分】
                              │
              ┌───────────────┴───────────────┐
              ↓                               ↓
        【主料采购】                    【辅料采购】
              │                               │
              └──────────┬────────────────────┘
                         ↓
                  【按供应商拆分】
                         │
         ┌────────────────┴────────────────┐
         ↓                                  ↓
   【一张订单 → 多张采购单】      【多张订单 → 合并成一张采购单】
   （同一供应商只出一张PO）      （不同订单同一供应商合并）
```

### 3.2 订单状态流转

| 状态 | 说明 | 触发动作 |
|------|------|----------|
| 已创建 | 订单录入完成 | 销售录入 |
| 已确认 | 客户确认，订单生效 | 销售确认 |
| **已拆分** | 订单已拆分为采购单 | **自动触发采购** |
| 采购中 | 采购单正在处理 | 采购确认 |
| 已到货 | 主辅料到货 | 仓库确认 |
| 生产中 | 窗帘正在加工 | 生产开单 |
| 生产异常 | 面料质量问题/残次/米数不足 | **生产反馈** |
| 已完成 | 生产完成，待生成安装单 | 生产确认 |
| **安装单已生成** | 自动生成安装单，进入调度 | **自动触发** |
| 已发货 | 送货途中 | 物流发货 |
| 已安装 | 安装完成 | 安装工确认 |
| 已验收 | 客户验收通过 | 客户确认 |
| 已完成 | 订单完结 | 财务确认 |

**关键：订单确认后自动触发采购拆分流程；生产完成后自动生成安装单**

### 3.3 采购单拆分规则（核心逻辑）

#### 拆单逻辑（一张订单 → 多张采购单）

订单确认后，系统自动按以下规则拆单：

1. **按物料类型拆分**
   - 主料（面料）：窗帘布、窗纱、软包布、硬包布
   - 辅料：轨道、罗马杆、挂钩、绑带、铅坠、花边等

2. **按供应商拆分**
   - 每个供应商独立生成一张采购单
   - 一张订单可能拆出 3~5 张采购单（主料供应商 + 各辅料供应商）

3. **订单明细中的供应商信息**
   - 每个产品/物料有 `supplier_id` 字段
   - 采购单按 `supplier_id` 分组

#### 合单逻辑（多张订单 → 合并成一张采购单）

当多张订单都含有**同一供应商**的物料时，合并为一张采购单：

| 场景 | 处理 |
|------|------|
| 同一供应商，多张订单 | 合并成一张采购单，标注关联的订单号 |
| 同一供应商，同一订单 | 保留分开（因为可能不同交期） |
| 不同供应商 | 各自独立采购单 |

**合并规则：**
- 供应商相同 + 交期相近（±3天）→ 合并
- 供应商相同 + 交期不同 → 分开（按交期优先）

#### 采购单内容

每张采购单包含：
- 采购单号（自动生成：PO + 年月日 + 序号）
- 供应商信息
- 关联订单号（可能多个）
- 物料明细（产品编号、名称、规格、数量、单价）
- 预计到货日期
- 采购状态（待采购/已下单/部分到货/全部到货）

### 3.4 生产跟踪与质量反馈（新增）

#### 生产过程跟踪

在生产过程中（生产中状态），可能出现以下问题需要跟踪和反馈：

| 问题类型 | 说明 | 处理方式 |
|----------|------|----------|
| 面料质量问题 | 色差、瑕疵、纹路异常 | 拍照记录，触发采购补货 |
| 面料残次 | 有明显缺陷无法使用 | 记录残次数量，触发采购补货 |
| 米数不足 | 实际米数少于订单需求 | 记录差额，触发采购补货 |

#### 质量反馈流程

```
生产中发现问题 → 填写生产反馈单 → 通知销售/采购 → 补货/调货 → 继续生产
                                                    ↓
                                              影响交期？
                                              → 更新订单交期 + 通知客户
```

**生产反馈单内容：**
- 关联订单号 + 关联采购单号
- 问题类型（质量/残次/米数不足）
- 问题描述 + 照片
- 反馈时间 + 反馈人
- 处理状态（待处理/已处理/已解决）

#### 自动生成安装单

**触发条件：生产完成后（状态变为"已完成"）**

**安装单自动生成逻辑：**
1. 系统根据订单信息自动创建安装单
2. 安装单包含：客户信息、地址、窗户位置、产品明细、安装要求
3. 安装单进入调度池，等待分配安装工人

**安装单内容：**
- 安装单号（INS + 年月日 + 序号）
- 关联订单号
- 客户姓名 + 电话 + 地址
- 安装地址（可能与客户地址不同）
- 产品明细（窗帘位置、款式、数量）
- 测量数据摘要
- 安装要求（特殊工坊/注意事项）
- 预约安装日期（系统根据交期推算或手动指定）
- 安装状态（待分配/已分配/已安装/已验收）

### 3.5 安装单展示与通知（新增）

#### 安装信息展示

安装单生成后，系统向以下人员展示可安装信息：

| 角色 | 展示内容 | 展示方式 |
|------|----------|----------|
| 销售人员 | 安装时间、安装进度、客户验收状态 | 订单详情页 |
| 客户 | 安装日期、时间、联系方式、安装进度 | 微信通知/小程序 |
| 安装工人 | 客户地址、产品明细、安装要求 | 手机端/安装工APP |
| 店长/财务 | 安装安排一览表 | 调度看板 |

#### 安装单调度流程

```
安装单生成 → 店长分配安装工人 → 安装工人确认接单 → 按时上门安装
                                                    ↓
                                          安装完成 → 填写安装记录
                                                    ↓
                                          客户验收 → 订单完结
```

### 3.6 窗帘行业专属功能

#### 🔴 P0（必须做，MVP核心）

| 功能 | 说明 |
|------|------|
| **多计算方式** | 支持：按米、按平方、按窗、固定价，系统自动根据产品类型选计算方式 |
| **自动算料** | 根据窗户尺寸（高×宽）+ 褶皱倍数 + 幅宽，自动计算面料米数 |
| **测量管理** | 记录测量数据（窗户尺寸、墙面状况、特殊需求），拍照上传 |
| **安装调度** | 分配安装工人、预约安装时间、跟踪安装进度 |
| **辅料关联** | 每个窗户位置关联对应辅料（轨道型号、挂钩类型等） |

#### 🟡 P1（重要，第二批）

| 功能 | 说明 |
|------|------|
| **多门店支持** | 门店切换、门店库存、门店销售统计 |
| **员工角色细分** | 导购、测量员、安装工、店长、财务不同权限 |
| **照片墙** | 测量照片、施工照片、验收照片绑定订单 |
| **微信通知** | 订单状态变更自动推送给客户微信 |
| **快速报价单** | 输入测量数据 → 一键生成带图报价单 → 打印/发微信 |

#### 🟢 P2（优化，第三批）

| 功能 | 说明 |
|------|------|
| **面料用量模拟** | 根据不同幅宽模拟几种下料方案，选最优 |
| **库存预警** | 面料/辅料库存低于阈值自动提醒 |
| **数据导入** | 从Excel批量导入客户、产品、订单数据 |
| **财务报表** | 按门店/按员工/按时间段统计销售额、利润 |

## 四、技术改造方向

### 4.1 数据模型改造

```
Customer（客户）
  ├── 测量记录（MeasureRecord）← 新增
  │     ├── 窗户位置
  │     ├── 高/宽
  │     ├── 幅数
  │     ├── 照片
  │     └── 备注
  ├── 订单（Order）
  │     ├── 订单状态（改造：新增多个状态节点）
  │     ├── 订单明细（OrderItem）← 改造
  │     │     ├── 产品类型（窗帘/窗纱/软包/硬包）
  │     │     ├── 计算方式（按米/按窗/按平方）
  │     │     ├── 面料用量（自动算出的）
  │     │     ├── 物料类型（主料/辅料）← 新增
  │     │     ├── 供应商ID（supplier_id）← 新增
  │     │     ├── 辅料明细（绑定关联）
  │     │     └── 安装信息
  │     ├── 采购单列表（PurchaseOrder）← 新增（关联）
  │     └── 安装记录（InstallationRecord）← 新增
  │           ├── 安装工
  │           ├── 预约时间
  │           ├── 实际完成时间
  │           └── 照片
  └── 跟进记录（FollowUpRecord）

PurchaseOrder（采购单）← 新增
  ├── 采购单号（PO + 年月日 + 序号）
  ├── 供应商ID
  ├── 关联订单ID列表（可能有多个）
  ├── 物料明细（JSON）
  ├── 预计到货日期
  ├── 采购状态
  └── 创建时间

Supplier（供应商）← 已有，需确认字段完整
  ├── 供应商名称
  ├── 联系人/电话
  ├── 地址
  └── 物料类型（主料供应商/辅料供应商）
```

- 后续如果要上微信功能，可扩展
