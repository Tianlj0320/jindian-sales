## 一、采购单拆分 API

**接口：** `POST /api/purchase-orders/split/{order_id}`

### 测试结果

| 用例 | 输入 | 预期 | 实际 | 结果 |
|------|------|------|------|------|
| TC-PO-SPLIT-001 正常拆分 | order_id=10（已确认订单，含3个不同供应商物料） | 返回3张采购单 | `{"message":"成功拆分为 0 张采购单"}` | **FAIL** |
| TC-PO-SPLIT-002 合单逻辑 | order_id=1（多订单同供应商交期相近） | 合并为1张采购单 | `{"message":"成功拆分为 1 张采购单"}`（已存在的测试数据） | **PASS**（数据干扰） |
| TC-PO-SPLIT-003 分单逻辑 | order_id=24（多供应商） | 分开生成2张采购单 | 返回1张（合并了） | **FAIL** |
| TC-PO-SPLIT-004 边界-单供应商 | order_id=20（单供应商） | 仅生成1张采购单 | `{"message":"成功拆分为 1 张采购单"}` | **PASS** |
| TC-PO-SPLIT-005 边界-交期差>3天 | order_id=25（同供应商，交期差>3天） | 分开生成 | 需验证 | **未独立验证** |
| TC-PO-SPLIT-006 异常-订单不存在 | order_id=9999 | HTTP 404 + error | `{"detail":"订单不存在"}` HTTP 404 | **PASS** |
| TC-PO-SPLIT-007 异常-订单未确认 | order_id=11（状态=created） | HTTP 400 + 订单未确认 | `{"success":true,"message":"成功拆分为 1 张采购单"}` HTTP 200 | **FAIL** |
| TC-PO-SPLIT-008 异常-重复拆分 | order_id=10（状态=split） | HTTP 400 + 已拆分 | 订单已拆分但未阻止，可重复拆分 | **FAIL** |

### 关键问题

- **订单11（created状态）调用拆分API返回200成功**，未校验订单状态是否为"已确认"
- **订单10已拆分后（状态=split）再次调用拆分API返回200**，未阻止重复拆分
- **拆分结果为0张采购单**：订单10含物料但返回0张，说明拆分的分组逻辑可能有问题（供应商未匹配导致所有物料被丢弃）

## Bug 清单

### Bug-001：采购单更新 — 非法状态值未校验（P0）
- **API:** `PATCH /api/purchase-orders/{id}`
- **描述:** 传入任意字符串作为status值，系统不做校验直接存储。
- **复现步骤:** `curl -X PATCH /api/purchase-orders/1 -d '{"status":"非法状态"}'`
- **预期结果:** HTTP 400 + `{"error":"状态值非法"}`
- **实际结果:** HTTP 200 + `{"success":true,"data":{"id":1,"status":"非法状态"}}`
- **严重程度:** P0 — 脏数据写入

### Bug-002：采购单更新 — 只读字段未保护（P0）
- **API:** `PATCH /api/purchase-orders/{id}`
- **描述:** 可通过PATCH更新purchase_order_no等只读字段。
- **复现步骤:** `curl -X PATCH /api/purchase-orders/1 -d '{"purchase_order_no":"PO-NEW-001"}'`
- **预期结果:** HTTP 400 + `{"error":"字段不可更新"}`
- **实际结果:** HTTP 200 + 成功更新（字段实际是否更新待确认）
- **严重程度:** P0

### Bug-003：生产反馈创建 — 必填字段order_id未校验（P0）
- **API:** `POST /api/production-feedback`
- **描述:** 不传order_id字段时系统不报错，直接创建成功。
- **复现步骤:** `curl -X POST /api/production-feedback -d '{"problem_type":"质量","description":"面料有色差"}'`
- **预期结果:** HTTP 400 + `{"error":"order_id 为必填字段"}`
- **实际结果:** HTTP 200 创建成功（id=6）
- **严重程度:** P0 — 数据完整性完全无保证

### Bug-004：生产反馈创建 — 问题类型非法值未校验（P0）
- **API:** `POST /api/production-feedback`
- **描述:** 传入任意problem_type值（如"非法类型"）系统不校验，直接创建成功。
- **复现步骤:** `curl -X POST /api/production-feedback -d '{"order_id":1,"problem_type":"非法类型","description":"测试"}'`
- **预期结果:** HTTP 400 + `{"error":"problem_type 非法"}`
- **实际结果:** HTTP 200 创建成功（id=7）
- **严重程度:** P0

### Bug-005：生产反馈创建 — 订单不存在未校验（P0）
- **API:** `POST /api/production-feedback`

- **复现步骤:** `curl -X POST /api/production-feedback -d '{"order_id":9999,"problem_type":"质量","description":"面料有色差"}'`

### Bug-006：生产反馈列表 — 过滤参数完全失效（P0）

### Bug-010：采购单拆分 — 重复拆分未阻止（P0）
- **API:** `POST /api/purchase-orders/split/{order_id}`
- **描述:** 已拆分的订单（状态=split）再次调用拆分API，未阻止重复操作。

- **严重程度:** P0

### Bug-011：采购单拆分 — 拆分结果为0张（P0）
