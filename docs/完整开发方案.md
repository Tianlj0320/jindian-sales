# 金典软装 · 窗帘店全流程管理系统开发方案

> 版本：v1.0 | 基于 Node.js + NestJS + Prisma + PostgreSQL + Docker + Git
> 覆盖：销售 → 采购 → 加工 → 安装 → 售后 全链路

---

### 为什么选这套架构

| 需求 | 方案 | 原因 |
|------|------|------|
| 数据库在云端 | 阿里云 RDS PostgreSQL | 门店PC+外出手机都能访问 |
| 随处办公 | API 部署在云服务器 | 有网就能用，C/S变B/S |
| 开发效率 | NestJS + Prisma | 类型安全，自动生成类型定义 |
| 部署一致性 | Docker | 本地开发环境和线上一致 |
| 版本管理 | Git | 代码可追溯，团队协作 |

### 2.1 基础资料模块

```prisma
// 产品
model Product {
  id          Int      @id @default(autoincrement())
  code        String   @unique          // 产品编码
  name        String                     // 产品名称
  category    String                     // 分类：布帘/纱帘/卷帘/配件
  subCategory String?                    // 子分类：高精密/棉麻/涤纶
  type        String   @default("面料")   // 类型：面料/辅料/成品
  calcType    String   @default("per_meter") // 计价方式：按米/平方/窗/固定
  unit        String   @default("米")     // 单位
  width       Float?                      // 门幅(cm)
  standardWidth Float? @default(0)        // 门幅(m)
  color       String?                     // 颜色
  pattern     String?                     // 花型
  material    String?                     // 材质成分
  price       Float                       // 零售价
  costPrice   Float?                      // 成本价
  minPrice    Float?   @default(0)        // 最低售价
  stock       Float    @default(0)        // 库存
  safetyStock Float    @default(0)        // 安全库存
  imagePath   String?                     // 产品图片
  isActive    Boolean  @default(true)     // 上架状态
  remark      String?                     // 备注
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
  supplierId  Int?                        // 供应商
  categoryId  Int?                        // 分类
  orderItems  OrderItem[]
}
```

```prisma
  roundAmount     Float    @default(0)     // 抹零金额

  received        Float    @default(0)     // 已收款(累计)
  debt            Float    @default(0)     // 欠款

  statusKey       String   @default("initial") // 状态key (12态)
  statusLabel     String   @default("待量尺")
  statusColor     String   @default("#909399")

  installDate     DateTime?
  installTimeSlot String?
  
  // 内容

  remark          String?

model OrderItem {
  id            Int      @id @default(autoincrement())

  product       Product? @relation(fields: [productId], references: [id])
  productName   String
  productCode   String?

  // 规格尺寸(窗帘核心)

  width         Float    @default(0)         // 窗宽(m)
  height        Float    @default(0)         // 窗高(m)
  foldRatio     Float    @default(2.0)       // 褶皱倍数
  materialQty   Float    @default(0)         // 实际用料(自动计算)
  unit          String   @default("米")       // 单位
  
  // 计价
  calcType      String   @default("per_meter") // 计价方式
  unitPrice     Float    @default(0)          // 单价
  qty           Float    @default(1)          // 数量
  discount      Float    @default(1.0)        // 折扣率
  amount        Float    @default(0)          // 金额
  finalAmount   Float    @default(0)          // 折后金额
  
  // 工艺
  openType      String?                       // 开合方式
  styleCode     String?                       // 款式编码
  processDesc   String?                       // 工艺描述
```

| Production 生产 | 5 | 反馈CRUD + 统计 |
| Finance 财务 | 8 | 应收/应付/费用/摘要 |
| Report 报表 | 6 | 销售汇总/排行/库存预警/利润 |
| System 系统 | 5 | 字典/日志/门店配置 |

### 3.2 核心业务流程API时序

**完整订单流程：**
```
POST   /orders                    → 创建订单
GET    /orders/{id}               → 查看订单详情
POST   /orders/{id}/advance       → 推进状态
         [confirmed → split]      → 自动拆分采购单
POST   /orders/{id}/receive       → 收款登记
GET    /orders/{id}/purchases     → 查看关联采购单
GET    /orders/{id}/installations → 查看关联安装单
```

**采购流程：**
```
GET    /purchases                  → 采购单列表
POST   /purchases                  → 手动创建采购单
PUT    /purchases/{id}/status      → 更新状态
POST   /purchases/{id}/receive     → 到货入库
```

## 五、开发和部署步骤

### Phase 1: 项目初始化（2-3天）

```bash
# 1. 创建 NestJS 项目
npm i -g @nestjs/cli
nest new curtain-system

# 2. 安装 Prisma
cd curtain-system
npm install @prisma/client
npm install -D prisma
npx prisma init

# 3. 配置数据库连接 (修改 .env)
# DATABASE_URL="postgresql://admin:password123@你的阿里云RDS地址:5432/curtain_db"

# 4. 编写 Prisma Schema（按上述完整模型）
npx prisma migrate dev --name init
npx prisma generate

# 5. 初始化 Git
git init
git add .
git commit -m "初始化项目骨架"
```

### Phase 2: 核心业务模块（2-3周，可并行开发）

按模块依次开发，每个模块包含：Model → Service → Controller → DTO → 测试

```
第一周: Auth模块 + 客户管理 + 产品管理
第二周: 订单管理(核心) + 状态机
第三周: 采购 + 仓库 + 生产反馈 + 安装
```

### Phase 3: 扩展模块（1周）

```
财务模块 + 报表统计 + 字典/日志/系统设置
```

### Phase 4: 前端开发（3-4周，与后端并行）

```
第一周: PC管理后台(Vue3) + 微信小程序骨架
第二周: 客户/产品页面 + 订单页面
第三周: 采购/仓库/安装页面 + 小程序核心功能
第四周: 财务/报表 + 客户查进度页面
```

### Phase 5: Docker容器化部署

```dockerfile
# Dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
RUN npm run build
EXPOSE 3000
CMD ["node", "dist/main"]
```

```yaml
# docker-compose.yml
version: '3.8'
services:
  api:
    build: .
    ports:
      - "3000:3000"
    environment:
      DATABASE_URL: postgresql://admin:password123@阿里云RDS地址:5432/curtain_db
      JWT_SECRET: your-secret-key
    restart: always
```

### Phase 6: 部署上线

```bash
# 1. 打包Docker镜像
docker build -t curtain-api .

# 2. 推送到阿里云镜像仓库
docker tag curtain-api registry.cn-hangzhou.aliyuncs.com/你的仓库/curtain-api:latest
docker push registry.cn-hangzhou.aliyuncs.com/你的仓库/curtain-api:latest

# 3. 在阿里云ECS上拉取并运行
ssh root@你的ECS
docker pull registry.cn-hangzhou.aliyuncs.com/你的仓库/curtain-api:latest
docker run -d -p 3000:3000 --name curtain-api --restart always curtain-api

# 4. 配置Nginx反向代理 + SSL证书
```
