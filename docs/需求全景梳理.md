## 三、与原始 NestJS 系统的差异对比

原始系统(D:\JD\curtain-backend\) 的技术栈是 NestJS + Prisma + PostgreSQL，数据模型如下：

| 表 | 原始字段 | 当前系统对比 |
|----|---------|-------------|
| Product | name, code, category, width, unit, price, costPrice, material, **imagePath**, stock | ✅ 已覆盖，缺 imagePath |
| Customer | name, **phone(unique)**, address, **type(散客/设计师/装修公司)** | ✅ 已覆盖，unique约束+type字段有但前端未展示 |
| Order | orderNo, customerId, **status(待测量→待下料→生产中→已完工→已交付)**, totalAmount, **paidAmount**, remark, **storeId** | 状态机更丰富(12态)，缺storeId，paidAmount命名不同 |
| OrderItem | location, width, height, foldRatio, **materialQty**, subtotal | ✅ 已覆盖，缺materialQty独立字段 |
| Dictionary | type, label, value, sortOrder, isActive | ✅ 已覆盖，缺前端管理页面 |
