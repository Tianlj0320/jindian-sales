## 一、行业 ERP/软装管理系统

### 1.1 三维家（3vone.it）

| 维度 | 情况 |
|------|------|
| 定位 | 全屋定制设计+生产一体化 |
| 价格 | 年费制，橱柜/衣柜为主 |
| 窗帘模块 | 有，但非核心，主攻全屋定制 |
| 优点 | 设计可视化强，设计稿直接对接生产 |
| 缺点 | 贵，年费动辄上万；窗帘定制功能弱；操作复杂，店员学习成本高 |
| 适合 | 大型全屋定制工厂 |

**结论**：❌ 过度设计，不适合金典软装

## 二、自建 vs 购买决策

| 维度 | 自建（当前方案） | 购买通用 ERP |
|------|---------------|-------------|
| 初始成本 | 低（时间成本） | 高（年费几千~几万） |
| 维护成本 | 低（完全可控） | 中（等供应商排期） |
| 行业贴合度 | 高（按金典实际需求） | 低（通用功能多） |
| 移动端 | 好（自建响应式） | 差（很多无移动端） |
| 数据安全 | 高（在自己服务器） | 中（第三方托管） |
| 扩展性 | 高（随时加功能） | 低（受制于产品规划） |
| 适合规模 | 月均50-200单 | 月均500单以上 |

**金典软装选择自建的理由**：
- 月均100单，需求简单，通用ERP 80%的功能用不上
- 老板有计算机背景，能自己维护
- 小红书/抖音获客为主，需要和营销系统打通，通用ERP做不到
- 长期主义：自建系统是资产，买的软件是费用

## 三、窗帘行业特殊需求参考

### 3.1 窗帘订单复杂度模型

```
普通窗帘单：面料+尺寸+加工费  →  简单
复杂窗帘单：面料+配件+罗马杆+安装+多窗户  →  中等
全屋软装单：窗帘+墙纸+硬包+岩板  →  复杂
```

**金典软装定位**：中高端定制，主要接复杂窗帘单和全屋软装单

### 3.2 窗帘行业几个关键坑

1. **门幅问题**：定高布（宽幅固定280cm，按米计价）和定宽布（按窗宽分格裁剪）计价方式不同，系统需区分
2. **损耗问题**：窗帘布料有损耗，报价时需考虑（一般加10-15%）
3. **安装延期**：窗帘定制生产周期7-15天，安装延期是客户投诉重灾区，需安装日期跟踪
4. **欠款问题**：软装行业赊账严重，欠款管理是核心
5. **导购折扣权限**：不同级别导购折扣不同，需在系统里控制

### 3.3 窗帘色差问题

窗帘是染布，不同批次有色差。行业通常做法：
- 大货前先打样确认
- 系统里备注批次/缸号
- 库存按批次管理（非当前系统范围）

## 四、行业趋势（2026年参考）

### 4.1 智能窗帘加速渗透

- 2026年智能窗帘渗透率估计达28%（受益于全屋智能）
- 超静音电机成标配（40dB以下）
- 系统需支持电动窗帘订单（电机+轨道+帘布分开报价）

### 4.2 小红书种草 → 私域转化

窗帘消费决策链：
```
小红书/抖音刷到 → 被内容种草 →私信/评论咨询 → 加微信 → 到店/上门量尺 → 下单
```

**系统设计启示**：客户来源统计很重要，小红书来的客户和自然到店客户ROI完全不同

### 4.3 奶油风/中古风带火高精密窗帘

- 高精密窗帘（遮光好+垂坠感强）成主流
- 颜色：奶咖/米灰/浅绿
- 面料：雪尼尔、棉麻、高精密
- 系统产品资料需要支持面料类型筛选
