# 金典软装销售系统 V4.0 — 系统概述

## 1. 项目背景

金典软装是一家专注于窗帘销售与安装服务的软装公司。原销售系统 V3.0 已稳定运行多年，积累了丰富的业务数据。随着业务规模扩大，系统面临以下挑战：

- **订单流程割裂**：销售订单、安装单、采购单各自独立，缺乏统一关联
- **财务数据滞后**：应收应付依赖人工统计，无法实时反映经营状况
- **库存管理粗放**：无法精确追踪布料配件的进出库流向
- **数据孤岛**：客户、产品、订单、安装、仓库、财务之间缺乏有效打通

V4.0 是对原系统的全面重构，旨在建立一个**以订单为中心**的全链路数字化管理平台。

---

## 2. V4.0 升级目标

| 目标 | 说明 |
|------|------|
| 全链路追溯 | 从客户咨询 → 订单 → 生产 → 采购 → 安装 → 售后，全流程可查 |
| 财务一体化 | 订单应收、安装应付、采购应付实时联动，财务报表自动生成 |
| 库存精准化 | 支持多仓库、批次管理、库存预警与流转记录 |
| 安装协同化 | 安装工/安装队统一管理，安装单与订单双向关联 |
| 报表决策化 | 提供经营分析、客户分析、产品分析等多维度报表 |

---

## 3. 技术栈

### 后端

- **框架**：FastAPI（Python 3.10+）
- **ORM**：SQLAlchemy 2.0（原生 SQLAlchemy）
- **数据库**：SQLite（开发/演示）、PostgreSQL（生产推荐）
- **认证**：JWT（HS256）
- **API**：RESTful + OpenAPI 3.0 自动文档

### 前端

- **技术**：原生 JavaScript（ES6+）+ HTML5 + CSS3
- **无框架依赖**：保持轻量，优先顺序表和简单组件
- **HTTP 客户端**：Fetch API

### 部署

- **推荐**：Docker Compose 一键部署
- **反向代理**：Nginx（生产环境）

## 4. 系统定位

```
┌─────────────────────────────────────────────┐
│              金典软装销售系统 V4.0            │
│                                             │
│  客户管理  │  销售订单  │  安装管理           │
│  产品管理  │  采购管理  │  仓库管理           │
│  财务管理  │  数据报表  │  系统设置           │
└─────────────────────────────────────────────┘
```

V4.0 是面向公司内部运营的管理系统，不直接面向终端消费者（CRM 侧由线下或第三方负责）。
