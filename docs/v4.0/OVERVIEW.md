# 金典软装ERP V4.0 — 系统概述

## 1. 项目背景

金典软装是一家专注于窗帘销售与安装服务的软装公司。原销售系统 V3.0 已稳定运行多年，积累了丰富的业务数据。随着业务规模扩大，V3.0 在订单流程、财务数据、库存管理等方面已无法满足日益增长的业务需求。

V4.0 是对原系统的全面重构，采用前后端分离架构，旨在建立一个**以订单为中心**的全链路数字化管理平台。

---

## 2. V4.0 升级目标

| 目标 | 说明 |
|------|------|
| 全链路追溯 | 从客户咨询 → 订单 → 采购 → 加工 → 安装 → 售后，全流程可查 |
| 财务一体化 | 订单应收、采购应付实时联动，自动生成财务报表 |
| 库存精准化 | 采购入库自动更新库存，支持多仓库管理 |
| 安装协同化 | 安装工/安装队统一管理，安装单与订单自动关联 |
| 操作可追溯 | 全量操作日志记录，关键操作可审计 |
| 字典可配置 | 28 种字典类型、~120 个字典项，支持运行时热配置 |

## 3. 技术栈

### 后端

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| 框架 | FastAPI | Python 3.12+，异步高性能 |
| ORM | SQLAlchemy 2.0 | 异步原生，支持 Type Hint |
| 数据库 | SQLite + aiosqlite | 开发/演示环境，文件即数据库 |
| 认证 | JWT (HS256) | 72h 过期，Bearer Token |
| API 文档 | OpenAPI 3.0 | FastAPI 自动生成 /docs |
| 数据验证 | Pydantic v2 | 请求/响应模型校验 |
| 生产推荐 | PostgreSQL | 可无缝切换 |

### 前端

| 组件 | 技术选型 |
|------|----------|
| 框架 | Vue 3 (Composition API) |
| 构建工具 | Vite |
| UI 组件库 | Element Plus |
| 路由 | Vue Router 4 (Hash History) |
| 状态管理 | Pinia |
| HTTP 客户端 | Axios |

### 部署

| 环境 | 方案 |
|------|------|
| 开发 | Uvicorn hot-reload |
| 生产推荐 | Docker Compose + Nginx 反向代理 |
| 数据库文件 | 持久化至 data/ 目录 |

## 4. 系统功能全景

```
金典软装ERP V4.0
│
├── 工作台（Dashboard）
│   └── 今日概况、月度销售、待办提醒、产品排行
│
├── 订单管理（核心模块）
│   ├── 订单列表（多条件筛选、分页）
│   ├── 订单创建（含明细行、自动计算）
│   ├── 订单详情（关联采购/安装/财务信息）
│   ├── 状态机流转（11 个状态，支持回滚）
│   └── 智能采购分单（跨单合并）
│
├── 基础资料
│   ├── 产品管理（分类树、加工类型、库存关联）
│   ├── 客户管理（含跟进记录）
│   ├── 供应商管理（含付款信息）
│   ├── 员工管理
│   ├── 角色权限（JSON 权限配置）
│   └── 加工类型（含辅料规则）
│
├── 采购管理
│   ├── 采购单列表/创建/详情
│   └── 采购入库（自动更新库存、生成库存流水）
│
├── 仓库管理
│   ├── 库存查询（含低库存预警）
│   ├── 库存流水（全量追溯）
│   └── 多仓库管理
│
├── 安装管理
│   ├── 安装单创建与分配
│   ├── 安装队/安装工管理
│   └── 安装状态更新（含照片上传）
│
├── 生产反馈
│   ├── 品质问题反馈
│   └── 反馈处理与跟踪
│
├── 财务管理
│   ├── 应收管理（按订单自动生成）
│   ├── 应付管理（按采购单自动生成）
│   └── 日常费用记录
│
└── 系统设置
    ├── 店铺信息配置
    ├── 字典管理（类型 + 项 CRUD）
    └── 操作日志查询
```
