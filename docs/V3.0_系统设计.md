## 三、订单状态机

### 3.1 状态定义

| status_key | 中文状态 | 说明 |
|------------|----------|------|
| `created` | 已创建 | 订单录入完成，等待确认 |
| `confirmed` | 已确认 | 客户确认，订单生效，自动触发采购拆分 |
| `split` | 已拆分 | 订单已拆分为采购单 |
| `purchasing` | 采购中 | 采购单正在处理 |
| `arrived` | 已到货 | 主辅料到货 |
| `producing` | 生产中 | 窗帘正在加工 |
| `production_exception` | 生产异常 | 面料质量问题/残次/米数不足 |
| `production_done` | 已完成 | 生产完成，待生成安装单 |
| `installation_generated` | 安装单已生成 | 自动生成安装单，进入调度 |
| `shipped` | 已发货 | 送货途中 |
| `installed` | 已安装 | 安装完成 |
| `accepted` | 已验收 | 客户验收通过 |
| `closed` | 已完成 | 订单完结 |

## 四、采购单拆分流程

### 4.1 触发条件

**触发时机：** 订单状态由 `created` → `confirmed`（销售确认订单）

**触发入口：** `/api/v3/orders/{order_no}/confirm`

### 4.2 拆分流程图

```
┌─────────────────────────────────────────────────────────────┐
│                     订单确认触发                              │
│           orders.status_key = 'confirmed'                    │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                  Step 1: 收集订单明细                        │
│                                                             │
│  读取 order_items（包含 supplier_id, material_type）        │
│  判断是否已有独立表（V3.0新增），若无则从 items JSON 解析    │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│              Step 2: 按供应商+物料类型分组                   │
│                                                             │
│  分组Key = supplier_id + material_type                     │
│                                                             │
│  例：订单含 3 种产品                                        │
│  ┌─────────────────────────────────────────┐                │
│  │ 产品A → 供应商S1 + 主料  ──┐           │                │
│  │ 产品B → 供应商S1 + 主料  ──┤→ 合并    │                │
│  │ 产品C → 供应商S2 + 辅料  ──┘           │                │
│  └─────────────────────────────────────────┘                │
│  拆分结果：2张采购单（PO1: 供应商S1主料，PO2: 供应商S2辅料） │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│           Step 3: 检查合并规则（跨订单合单）                  │
│                                                             │
│  查询条件：                                                  │
│  - 同一供应商（supplier_id相同）                            │
│  - 交期相近（expected_date ± 3天内）                       │
│  - 状态为"待采购"或"已下单"的采购单                         │
│                                                             │
│  若满足条件 → 合并到已有采购单，追加items                   │
│  若不满足条件 → 新建采购单                                  │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│            Step 4: 生成采购单（purchase_orders）             │
│                                                             │
│  po_no = "PO" + 年月日 + 序号                               │
│  例：PO260421001                                             │
│                                                             │
│  字段赋值：                                                  │
│  - order_id: 当前订单ID                                     │
│  - supplier_id: 分组供应商ID                                │
│  - supplier_name: 供应商名称                                 │
│  - material_type: 主料/辅料                                 │
│  - status: '待采购'                                         │
│  - expected_date: 订单交期日期                               │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│           Step 5: 生成采购单明细（purchase_order_items）     │
│                                                             │
│  遍历分组内的 order_items                                   │
│  逐条写入 purchase_order_items                              │
│                                                             │
│  字段赋值：                                                  │
│  - purchase_order_id: 所属采购单ID                          │
│  - order_item_id: 关联订单明细ID                            │
│  - product_id: 产品ID                                       │
│  - qty: 采购数量                                            │
│  - unit_price: 单价                                         │
│  - subtotal: 小计                                           │
│  - status: '待发'                                           │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                Step 6: 更新订单状态                          │
│                                                             │
│  orders.status_key = 'split'                                │
│  orders.purchase_split_status = '已拆分'                   │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                       拆分完成                               │
│                                                             │
│  返回拆分结果（含采购单号列表）                              │
└─────────────────────────────────────────────────────────────┘
```

## 五、安装单流程图

### 5.1 自动生成触发条件

**触发时机：** 订单状态 `production_done`（生产完成）

**触发入口：** 生产确认操作后自动执行

### 5.2 安装单生命周期

```
┌──────────────────────────────────────────────────────────────┐
│                    安装单自动生成                              │
│              orders.status_key = 'production_done'            │

│                                                               │
│ 从 order_items 读取：                                        │
│ - 窗户位置、产品名称、数量                                    │
│ - measurement_ref（测量数据摘要）                            │
└──────────────────────────┬───────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────────┐
│ Step 2: 创建安装单                                            │
│                                                               │
│ installation_orders 表写入：                                  │
│ - ins_no: "INS" + 年月日 + 序号                              │
│ - order_id, order_no                                         │
│ - customer_name, customer_phone                              │
│ - install_address                                            │
│ - install_date（从订单复制或自动推算）                        │

│ - measurement_ref                                            │
└──────────────────────────┬───────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────────┐
│ Step 3: 更新订单状态                                          │
│                                                               │
│ orders.status_key = 'installation_generated'                  │
│ orders.installation_order_status = '已生成'                    │
└──────────────────────────┬───────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────────┐
│ Step 4: 进入调度池                                            │
│                                                               │
│ 店长/管理员在【安装调度】页面查看待分配安装单                 │

└──────────────────────────┬───────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────────┐
│ Step 5: 分配安装工人                                          │
│                                                               │
│ PUT /api/v3/installation-orders/{ins_no}/assign              │
│                                                               │
│ 更新字段：                                                    │
│ - installer_id                                               │
│ - installer_name                                             │
│ - install_date                                               │
│ - install_time_slot                                          │
│ - status = '已分配'                                          │
└──────────────────────────┬───────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────────┐
│ Step 6: 安装工人确认接单                                      │
│                                                               │
│ 安装工人APP收到通知，确认接单                                │
│ 状态保持 '已分配'（工人可见）                                │
└──────────────────────────┬───────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────────┐
│ Step 7: 按预约时间上门安装                                    │
│                                                               │
│ 安装完成后填写：                                              │
│ - completion_photos（安装照片）                              │
│ - customer_confirm（客户确认：通过/有遗留问题）               │

│ - collection_amount（实收金额）                             │
│ - completion_remark（完成备注）                             │
└──────────────────────────┬───────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────────┐

│                                                               │
│ PUT /api/v3/installation-orders/{ins_no}/complete            │
│                                                               │
│ 更新字段：                                                    │
│ - status = '已验收'                                          │
│ - completed_at = 当前时间                                    │
│ - collection_status（未收/部分/已清）                        │
└──────────────────────────┬───────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────────┐
```

## 六、采购单状态机

| status | 中文 | 说明 |
|--------|------|------|
| `待采购` | 待采购 | 拆分完成，等待采购员处理 |
| `已下单` | 已下单 | 采购员已向供应商下单 |
| `部分到货` | 部分到货 | 有物料到货，部分未到 |
| `全部到货` | 全部到货 | 所有物料到货完成 |

**流转：待采购 → 已下单 → 部分到货/全部到货**

## 七、生产反馈流程

```
生产中发现问题
      │
      ▼
填写生产反馈单
feedback_type: 质量/残次/米数不足
      │
      ▼
通知销售/采购
      │
      ├── 需补货 → 触发补货采购（新建PO）→ 继续生产
      │                    ↓
      │             影响交期？
      │             → 更新订单交期 + 通知客户
      │
      └── 无需补货 → 直接继续生产
                    （处理备注记录）
```
