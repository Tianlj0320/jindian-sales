## 1. 项目概述

### 1.1 项目背景
金典软装（杭州古墩路欧亚达家居广场）主营窗帘、墙纸墙布、硬包、岩板定制业务，月均订单约100单，现有3-5名员工。随着业务增长，纯手工记账和 Excel 管理模式已无法满足需求，亟需一套轻量级销售管理系统。

### 1.2 项目目标
- 打通「获客→跟进→成交→生产→安装→售后→复购」全链路
- 减少手工台账，降低出错率
- 支持手机端查询（客户/安装工/老板）
- 为未来数据化运营（爆款分析、客户画像）打下基础

### 1.3 定位
**轻量自建，实用优先**。月均100单规模市面通用 ERP 过于笨重，选择 Python FastAPI + Vue.js 自建，配合阿里云 ECS 部署，长期可控可改。

### 1.4 技术选型
| 层次 | 技术 |
|------|------|
| 后端 | Python FastAPI + SQLAlchemy + SQLite |
| 前端 | Vue.js 3（CDN版本，无需构建） + 原生 HTML/CSS |
| 手机端 | 三套独立 HTML（管理端/安装工端/客户查进度） |
| 认证 | JWT（72小时有效） |
| 部署 | 阿里云 ECS（Python uvicorn 启动） |

## 3. 订单流程

### 3.1 标准订单流（7态→8态）

```
[待确认] → [已确认] → [生产中] → [已发货] → [安装中] → [已完成]
                    ↓                      ↓
              [已取消]               [已退回]
```

V2.2 扩展为8态：
- `created` — 待确认
- `confirmed` — 已确认
- `producing` — 生产中
- `shipped` — 已发货
- `installing` — 安装中
- `completed` — 已完成
- `cancelled` — 已取消
- `returned` — 已退回

### 3.2 金额计算公式

```
应收金额 = 标价金额 - 优惠金额 - 抹零金额
欠款 = 应收金额 - 已收金额
```

折扣上限由员工 `max_discount` 字段控制（如0.75即最多7.5折）。

## 4. 数据规范

### 4.1 产品计量
- 窗帘布料默认单位：**米**
- 门幅（宽度）：cm，如 280cm
- 克重：g/㎡
- 分类：定高布 / 定宽布 / 配件

### 4.2 订单号编码规则
```
YYMMDD + 序号（每日从001开始）
例如：260419001 表示2026年4月19日第1单
```

### 4.3 供应商类型
- 成品（整品供应）
- 布艺（面料供应）
- 配件（轨道/罗马杆等）

### 4.4 客户类型
- 零售（散户）
- 工程（项目）
- 分销（渠道）

## 6. 非功能性需求

### 6.1 性能
- 首页加载 < 2秒（SQLite 查询优化，建立索引）
- 支持10年历史数据（年/月分区筛选）

### 6.2 安全
- JWT Token 72小时有效
- 员工密码 bcrypt 加密存储
- 安装工验证码5分钟有效

### 6.3 可用性
- 单机部署，无需内网穿透即可局域网访问
- SQLite 文件每日自动备份
